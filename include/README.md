# Заголовочные файлы (.h)

В этой директории содержатся все заголовочные файлы проекта, определяющие интерфейсы и структуры основных модулей.

## Ключевые файлы

### `config.h`
**Центральный файл конфигурации системы.** Содержит критически важные параметры:
*   Аппаратные настройки для T-Display (пины дисплея, кнопок, батареи).
*   Параметры TOTP (интервал генерации, длина кода).
*   Настройки сетевой безопасности (таймауты сессий, лимиты).
*   Конфигурация BLE (имена устройств, параметры безопасности).
*   Пути к зашифрованным файлам данных в файловой системе.
*   Параметры управления питанием и энергосбережения.

### Менеджеры (`*_manager.h`)
Определяют к��ассы, отвечающие за ключевые функции системы:
*   `display_manager.h`: Управляет всем выводом на дисплей, включая анимации, темы и отрисовку элементов.
*   `key_manager.h`: Отвечает за загрузку, сохранение, шифрование и управление секретными ключами TOTP.
*   `web_server.h`: Современный веб-сервер с многоуровневой аутентификацией, управлением сессиями и защищенными API.
*   `wifi_manager.h`: Интеллектуальное управление WiFi с автоматическим отключением для экономии энергии.
*   `pin_manager.h`: Система PIN-защиты с криптографическим хешированием и защитой от брутфорса.
*   `config_manager.h`: Менеджер конфигурации с поддержкой динамических настроек и безопасного хранения.
*   `ble_keyboard_manager.h`: Защищенная BLE реализация с аутентификацией, шифрованием и bonding.
*   `crypto_manager.h`: Криптографический движок с AES-256, PBKDF2 и управлением ключами.
*   `password_manager.h`: Система управления паролями с генерацией, шифрованием и безопасным хранением.
*   `splash_manager.h`: Управление пользовательскими экранами загрузки.
*   `battery_manager.h`: Продвинутый мониторинг батареи с калибровкой и энергосбережением.

### Дополнительные компоненты
*   `totp_generator.h`: Высокопроизводительный генератор TOTP с поддержкой различных алгоритмов.
*   `ui_themes.h`: Система тем с поддержкой кастомизации цветовых схем.
*   `animation_manager.h`: Движок анимаций для плавных переходов интерфейса.
*   `log_manager.h`: Система логирования с уровнями важности и ротацией.
*   `web_pages/`: Встроенные веб-страницы с современным UI/UX дизайном и адаптивной версткой.

### Архитектурные особенности
*   **Модульность:** Каждый компонент независим и может быть легко модифицирован.
*   **Безопасность:** Все менеджеры интегрированы с криптографической системой.
*   **Производительность:** Оптимизированы для работы на ограниченных ресурсах ESP32.
*   **Расширяемость:** Архитектура позволяет легко добавлять новые функции.
